# تشغيل ClaudeClaw على Railway — دليل خطوة بخطوة

هذا الدليل ينشر ClaudeClaw كـ **خدمة دائمة** على Railway باستخدام **اشتراك Claude (OAuth)**، مع:
مهام مجدولة (cron + heartbeat)، لوحة تحكم ويب، وبوت Discord/Slack.

---

## ⚠️ اقرأ هذا أولاً (مهم)

- **ClaudeClaw يشغّل `claude` كـ daemon بصلاحيات تنفيذ أوامر.** على سيرفر متصل بالإنترنت، أي خطأ في الإعداد = مخاطرة أمنية. ابدأ بأقل صلاحية ووسّع تدريجياً.
- **بيانات اعتماد اشتراكك (OAuth) ستوضع على Railway.** عاملها كسرّ: لا ترفعها على GitHub أبداً، استخدم متغيرات البيئة فقط.
- **الـ OAuth token قد ينتهي ويحتاج تجديداً.** نخزّن ملف الاعتماد كاملاً (فيه refresh token) في Volume دائم عشان `claude` يجدّد تلقائياً. لو انتهى التجديد بتحتاج تعيد تسجيل الدخول محلياً وتحدّث المتغير.
- **سؤال التوافق مع شروط Anthropic:** أنت تستخدم اشتراكك الخاص داخل بيئة Claude Code. راجع شروط الخدمة بنفسك قبل التشغيل المستمر على سيرفر بعيد.

---

## المتطلبات

- حساب [Railway](https://railway.app) (الخطة المدفوعة لأن الخدمة تشتغل 24/7).
- حساب GitHub (لرفع مجلد النشر).
- جهازك المحلي عليه Claude Code مثبّت ومسجّل دخول باشتراكك.
- (اختياري) توكنات Discord/Slack لو بتفعّل البوتات.

---

## الخطوة 1 — المستودع

المستودع يحتوي ملفات النشر فقط (سورس ClaudeClaw يُستنسخ تلقائياً وقت بناء Docker):

```
claudeclaw-railway/
├── Dockerfile        ← يستنسخ moazbuilds/claudeclaw وقت البناء
├── entrypoint.sh
├── railway.json
├── .dockerignore
└── README.md
```

إذا أنشأ لك Claude المستودع عبر GitHub، فهو جاهز. وإلا ارفعه يدوياً:

```bash
git init && git add -A && git commit -m "ClaudeClaw on Railway"
git remote add origin https://github.com/<حسابك>/claudeclaw-railway.git
git push -u origin main
```

> لا تضِف أي توكن أو بيانات اعتماد داخل الملفات — كلها تُضبط كمتغيرات بيئة على Railway.

---

## الخطوة 2 — جهّز توكن مصادقة Claude

على **جهازك المحلي** (مسجّل دخول بـ Claude Code، يحتاج اشتراك Pro/Max):

```bash
claude setup-token
```

ينتج توكن طويل الأمد (صالح ~سنة) يبدأ بـ `sk-ant-oat...`. هذا نضعه في متغير `CLAUDE_CODE_OAUTH_TOKEN` على Railway.

> ⚠️ **أمان:** عامل هذا التوكن كسرّ. لا تلصقه في محادثات أو ملفات أو commits — فقط في خانة Variables المشفّرة على Railway. لو انكشف، اسحبه (revoke) من console.anthropic.com وولّد غيره.

> بديل: لو تفضّل، استخدم `CLAUDE_CREDENTIALS_JSON` بمحتوى `~/.claude/.credentials.json` كاملاً بدلاً من التوكن. استخدم واحداً من الاثنين.

---

## الخطوة 3 — أنشئ المشروع على Railway

1. Railway → **New Project** → **Deploy from GitHub repo** → اختر `claudeclaw-railway`.
2. Railway يكتشف `Dockerfile` و`railway.json` تلقائياً.
3. لا تنشر بعد — اضبط المتغيرات والـ Volume أولاً (الخطوتان 4 و5).

---

## الخطوة 4 — اربط Volume دائم (إلزامي)

بدون Volume، Railway يمسح الحالة وبيانات الاعتماد عند كل deploy.

- في الخدمة → **Settings → Volumes → Add Volume**.
- **Mount path:** `/data`
- احفظ.

> كل شيء مهم (الاعتماد، الجلسات، المهام، اللوقات) يُخزَّن تحت `/data`.

---

## الخطوة 5 — اضبط متغيرات البيئة (Variables)

في الخدمة → **Variables**. أضف ما يلزمك:

### إلزامي (المصادقة + الداشبورد)

| المتغير | القيمة |
|---|---|
| `CLAUDE_CODE_OAUTH_TOKEN` | التوكن من `claude setup-token` (الخطوة 2) — يبدأ بـ `sk-ant-oat...` |
| `WEB_TOKEN` | توكن تختاره أنت لحماية الداشبورد، مثلاً ناتج `openssl rand -base64 32` |

> بديل المصادقة: `CLAUDE_CREDENTIALS_JSON` بدلاً من التوكن (واحد منهما يكفي).

> `PORT` يوفّره Railway تلقائياً — **لا تضبطه يدوياً**. السكربت يربط الداشبورد على `0.0.0.0:$PORT`.

### المهام المجدولة (heartbeat) — اختياري

| المتغير | القيمة |
|---|---|
| `HEARTBEAT_ENABLED` | `true` لتفعيل النبضات الدورية |
| `HEARTBEAT_INTERVAL` | الفاصل بالدقائق (مثلاً `30`) |

> مهام cron نفسها تُنشأ من **الداشبورد** بعد التشغيل (أو كملفات jobs). الـ heartbeat هو التشغيل الدوري التلقائي.

### Discord — اختياري

| المتغير | القيمة |
|---|---|
| `ENABLE_DISCORD` | `true` |
| `DISCORD_TOKEN` | توكن البوت من Discord Developer Portal |
| `DISCORD_ALLOWED_USER_IDS` | معرّفك (أو عدة معرّفات مفصولة بفواصل) — **إلزامي وإلا يرفض الإقلاع** |
| `DISCORD_LISTEN_CHANNELS` | (اختياري) معرّفات قنوات يرد فيها بدون منشن |

### Slack — اختياري

| المتغير | القيمة |
|---|---|
| `ENABLE_SLACK` | `true` |
| `SLACK_BOT_TOKEN` | `xoxb-...` |
| `SLACK_APP_TOKEN` | `xapp-...` (Socket Mode) |
| `SLACK_ALLOWED_USER_IDS` | معرّفك — **إلزامي** |
| `SLACK_LISTEN_CHANNELS` | (اختياري) معرّفات قنوات |

> **تحذير من تحديث v1.0.26:** قائمة `allowedUserIds` الفاضية تعني **منع الجميع**، والديمون **يرفض الإقلاع** لو فعّلت بوت بدون معرّف مسموح واحد على الأقل. لا تنسَ هذا المتغير.

## الخطوة 6 — انشر وافتح الداشبورد

1. اضغط **Deploy**. راقب **Deploy Logs** حتى تشوف:
   `Web UI: http://0.0.0.0:<port>` و `web dashboard token set from WEB_TOKEN`.
2. في **Settings → Networking → Generate Domain** لإنشاء رابط عام (Railway يربطه بـ `$PORT`).
3. افتح الداشبورد عبر:
   ```
   https://<your-app>.up.railway.app/?token=<WEB_TOKEN>
   ```
   (التوكن مطلوب لكل مسارات `/api/*` ما عدا `/api/health`.)

من الداشبورد تقدر: تضيف مهام cron، تتابع التشغيلات واللوقات، وتضبط مستوى الأمان والموديل.

---

## التحقق من الصحة

- `https://<your-app>.up.railway.app/api/health` يرجّع حالة سليمة بدون توكن.
- لو ظهر `Not logged in` في اللوق → `CLAUDE_CREDENTIALS_JSON` غير صحيح أو ناقص.
- لو الخدمة تعيد التشغيل بسبب بوت → غالباً نسيت `*_ALLOWED_USER_IDS`.
- لو الداشبورد يرجّع 401 → استخدم `?token=` الصحيح أو ترويسة `Authorization: Bearer`.

---

## ملاحظات أمنية وتشغيلية

- ابدأ بمستوى أمان **read-only** من الداشبورد، ووسّع فقط عند الحاجة.
- لا تشارك رابط الداشبورد مع التوكن مع أحد.
- لتحديث الأداة: `./setup.sh` ثم `git push` ويعيد Railway البناء.
- التكلفة على Railway حسب وقت التشغيل والذاكرة (خدمة 24/7).
